# Pasta `projetos`

Esta pasta contém recursos e modelos relacionados aos projetos de engenharia do repositório.

Estrutura proposta:

- `docs/` - documentação específica de projetos (modelos, instruções).
- `templates/` - modelos reutilizáveis (escopo, cronograma, planilhas).
- `exemplos/` - exemplos e projetos de demonstração (JSON/YAML).

Como usar

1. Ajuste/adicione templates e exemplos conforme necessário.
2. Ao finalizar alterações, faça commit usando seu nome e email de autoria (conforme solicitado):

   git -C "c:\\Users\\vicen\\Gerenciador-de-Projetos-de-Engenharia" config user.name "vicente"
   git -C "c:\\Users\\vicen\\Gerenciador-de-Projetos-de-Engenharia" config user.email "vicentedesouza762@gmail.com"

Sugestão de commit:

    git add projetos
    git commit -m "feat(projetos): adicionar estrutura inicial da pasta projetos"

Se desejar, empurre para seu fork/remote e abra um Pull Request para o repositório principal.
