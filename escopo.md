# Escopo do Projeto – Gerenciador de Projetos de Engenharia Civil

## Objetivos

* Desenvolver uma plataforma completa para planejamento e controle de obras civis.
* Permitir gerenciamento de cronogramas, orçamentos, materiais, tarefas, equipes e documentos técnicos.
* Disponibilizar acesso via Web e Mobile consumindo a mesma API.
* Melhorar a organização e comunicação interna das equipes de engenharia.

## Diferenciais

* Suporte a múltiplos usuários por projeto com permissões configuráveis.
* Chat interno para comunicação entre membros.
* Armazenamento centralizado de plantas e documentos.
* Experiência completa e consistente entre Web e Mobile.

## Justificativa

* Obras civis sofrem com desorganização, perda de dados e comunicação falha.
* O sistema busca centralizar e padronizar informações, reduzindo erros.
* Tema alinhado ao ambiente profissiona.

## Tecnologias Pretendidas

* **Back-end:** Python (API REST).
* **Front-end Web:** HTML e CSS.
* **Mobile:** Flutter.
* **Banco de Dados:** MySQL.
* **Autenticação:** A definir
* **Hospedagem:** A definir (AWS, Render, Railway, etc.).
